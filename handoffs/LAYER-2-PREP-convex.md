# Layer 2-PREP: Convex Setup

**Status:** Can run in PARALLEL with Layer 1
**Branch:** `feature/convex-prep`
**Dependencies:** None

## Goal
Initialize Convex project, create schema for users and linking MentraOS userIds, deploy to cloud.

## Context
This is prep work for Layer 2 that can run in PARALLEL while Layer 1 is being tested. You're setting up Convex as the database for Clairvoyant to store user data, preferences, and subscriptions.

## What to Build

### 1. Initialize Convex
```bash
npx convex init
```
This creates `convex/` folder and config files.

### 2. Create Schema (`convex/schema.ts`)
```typescript
import { defineSchema, defineTable } from "convex/server";
import { v } from "convex/values";

export default defineSchema({
  users: defineTable({
    mentraUserId: v.string(),
    createdAt: v.number(),
  }).index("by_mentra_id", ["mentraUserId"]),
  
  mentraLinks: defineTable({
    userId: v.id("users"),
    mentraUserId: v.string(),
    linkedAt: v.number(),
  }).index("by_mentra_id", ["mentraUserId"]),
});
```

### 3. Create User Mutations (`convex/users.ts`)
- `getByMentraId(mentraUserId)` - query to find user by MentraOS ID
- `create({ mentraUserId })` - mutation to create new user
- `getOrCreate({ mentraUserId })` - mutation with upsert logic

### 4. Deploy to Convex Cloud
```bash
npx convex deploy
```
Save the deployment URL it gives you.

### 5. Environment Setup
- Add `CONVEX_URL` to `.env` with your deployment URL
- Update `.env.example` to include `CONVEX_URL=`

## Acceptance Criteria
